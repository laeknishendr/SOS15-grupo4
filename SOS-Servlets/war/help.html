Manual de usuario

GET

Acción sobre lista

Si la base de datos tiene elementos, el servidor devolverá una lista de las entradas. Si la base de datos está vacía, devolverá una lista vacia.

Acción sobre recurso

Si en la base de datos hay un recurso cuya clave sea igual al nombre solicitado, el servidor devolverá dicho elemento. Si no lo hay, el servidor devolverá un error 404, NOT FOUND.

POST

Acción sobre lista

Si no se introduce ningún dato, el servidor devolverá un error 400, BAD REQUEST. Si el dato que se quiere introducir ya está en la base de datos, el servidor devolverá un error 409, CONFLICT. Si no, el servidor devolverá 200 OK, y guardará el dato en la base de datos.

Acción sobre recurso

Esta acción no está permitida, el servidor devolverá un error 405, METHOD_NOT_ALLOWED.

PUT

Acción sobre lista

Esta acción no está permitida, el servidor devolverá un error 405, METHOD_NOT_ALLOWED.

Acción sobre recurso

Si no se introduce ningún dato, el servidor devolverá un error 400, BAD REQUEST. Si el dato que se quiere introducir no está en la base de datos, el servidor devolverá un error 403, FORBIDDEN. Si no, el servidor devolverá 200 OK, y actualizará el dato en la base de datos.

DELETE

Acción sobre lista

El servidor devolverá 200 OK y borrará toda la base de datos.

Acción sobre recurso

Si el elemento no está en la base de datos, el servidor devolverá un error 404, NOT FOUND. Si no, el servidor devolverá 200 OK y borrará el elemento de la base de datos.
